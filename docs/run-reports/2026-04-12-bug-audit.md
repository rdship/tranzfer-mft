## FIXED IN THIS COMMIT

### C3 — LazyInitializationException in scheduled jobs
- `AnomalyDetectionService.detectAnomalies()` — added `@Transactional(readOnly = true)`
- `PartnerProfileService.rebuildProfiles()` — added `@Transactional(readOnly = true)`
- `FlowExecutionRecoveryJob.recoverStuckExecutions()` — added `@Transactional`
- Root cause: `@Scheduled` methods run outside HTTP request lifecycle, so OSIV doesn't apply. They need explicit `@Transactional`.

### C4 — Prometheus `/actuator/prometheus` not exposed (13 services)
- Added `micrometer-registry-prometheus` dependency (if missing)
- Ensured actuator endpoint exposure includes `prometheus`
- Without this fix, Grafana dashboards had zero metrics from the majority of the platform.

### C5 — Partner Portal nginx redirect loop
- Fixed `try_files` directive in partner-portal nginx config
- Was: infinite redirect cycle to `/index.html`
- Partner Portal at http://localhost:3002 was completely inaccessible.

### C6 — Redis config `localhost:6379` → `redis:6379`
- Fixed in: ftp-web-service, notification-service, as2-service (and any others found)
- Inside Docker network, `localhost` resolves to the container itself, not the Redis container.
- Rate limiting, caching, and session management were non-functional in affected services.

## CTO ACTION REQUIRED

### C1 — Alertmanager SMTP configured with placeholder `smtp.example.com`
**Severity:** P0 — the platform is flying blind.
**Evidence:** `mft-alertmanager` logs show `smtp.example.com: no such host` and webhook to `:9095` connection refused. All alerts (PlatformMajorOutage, HighErrorRate, UnauthorizedAccess) are silently dropped.
**Fix:** Configure real SMTP credentials or a webhook endpoint (Slack, PagerDuty, OpsGenie) in `config/alertmanager/alertmanager.yml`.

### C2 — Missing DB column `p95latency_ms` in `metric_snapshots` table
**Severity:** P0 — generates hundreds of Postgres errors per minute.
**Evidence:** `ERROR: column ms1_0.p95latency_ms does not exist` on every analytics query.
**Root cause:** Hibernate entity `MetricSnapshot` maps `p95latencyMs` and `p99latencyMs` fields, but the Flyway migration that creates these columns doesn't exist (or was skipped — Flyway reports schema version 999 > latest migration 53).
**Fix:** Add a Flyway migration: `ALTER TABLE metric_snapshots ADD COLUMN p95latency_ms DOUBLE PRECISION, ADD COLUMN p99latency_ms DOUBLE PRECISION;`

### C7 — Slack connector TLS certificate failure
**Severity:** P2
**Evidence:** `PKIX path building failed: unable to find valid certification path to requested target`
**Fix:** Import Slack's CA certificate into the JVM truststore, or if behind a corporate proxy, import the proxy's CA cert: `keytool -importcert -file slack-ca.pem -keystore $JAVA_HOME/lib/security/cacerts -alias slack-ca`

### C8 — 4 file transfers permanently stuck on missing encryption keys
**Severity:** P2
**Evidence:** `Non-retryable failure (ENCRYPTION_KEY)` for files: `invoice_20250401.edi`, `partner-ack.mdn` (x2), `payroll.xml`
**Fix:** Either provision the missing encryption keys in keystore-manager, or mark these transfers as failed/cancelled to stop the retry spam.

### W1 — PGP Key Rotation scheduler misconfigured in as2-service
**Evidence:** `'Daily PGP Key Rotation Check' failed: No 'command' in task config`
**Fix:** Add the `command` field to the scheduler task configuration.

### W2 — Flyway schema version 999 > latest migration
**Evidence:** 3 services report this warning on startup. V999 is a "write-intents" sentinel migration that was inserted manually above the actual migration sequence. Any new migration with a version number < 999 will be ignored.
**Fix:** Rename V999 to a proper version number or use a Flyway repeatable migration (`R__write_intents.sql`).

### W3 — DMZ proxy receiving external scanner traffic
**Evidence:** `POST /reader-update` from public IPs every 14 seconds, upstream refuses.
**Fix:** See BUG-G in the earlier run report — port binding defaults to `0.0.0.0`.
